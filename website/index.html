Valid Blockchain — v0.7.0

Introducing
Three-Party Integrity.

The Valid Blockchain uses TPI, an original, lightweight consensus mechanism where three validators selected from an available pool verify each block by merit, not capital, stake, or tokens. Designed to run on modest hardware.

TPI is not a variant of PoS, PoW, or DPoS. Validator legitimacy is proven through participation, not wealth or compute.

Mechanism

How TPI Works

Three-Party Integrity is an original consensus mechanism, developed from scratch as a counter-reaction to unnecessarily heavy blockchain finality models. It does not borrow from PoS, PoW, or DPoS.

Random Selection

Three validators are randomly selected from the pool per block slot. No rotation schedule or delegation. Selection is unpredictable.

Merit-Based Production

Each selected validator independently computes a candidate block hash. The highest-merit validator among those in agreement produces the block.

2/3 Finality

The other two validators verify. Finality requires 2 of 3 in agreement, or 66%. Bad behavior is quarantined and loses standing. No capital is ever at risk as there is no stake.

Racer Backup

A racer backup system provides network resilience when selected validators fail to respond. Production continues without stalling the slot and always keeps perfect time.

Identity-Free Transport

As of v0.7.0, validator identity is no longer carried in the peer handshake. Peer connections are identity-free at the transport layer.

Arweave Archival

Every 2,160 blocks, a 6-hour archive segment is written and published to Arweave as permanent off-chain record. This is a response to the problem of ever-growing chain states.

Getting Started

Run a Validator

Requires Rust 1.70+. Install via rustup.rs.

bash 
# Clone and build
git clone https://github.com/HiImRook/accessible-pos-chain.git
cd accessible-pos-chain
cargo build --release

# Run a validator node
./target/release/pos-chain

See SETUP_GUIDE.md for full configuration and NETWORKING.md for peer and bootstrap details.
